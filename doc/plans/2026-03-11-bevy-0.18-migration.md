# Bevy 0.18 Migration Plan

## Overview

Migrate from Bevy 0.17.2 to Bevy 0.18, bumping all ecosystem dependencies (lightyear 0.26, avian3d 0.5, leafwing 0.20, bevy_common_assets 0.15, ron 0.12). Based on [research](../research/2026-03-11-bevy-0.18-migration.md).

## Current State Analysis

- Bevy 0.17.2 with lightyear 0.25.5, avian3d 0.4.1, leafwing 0.19, bevy_common_assets 0.14, ron 0.9
- Vendored git/ dependencies for bevy and lightyear (excluded from workspace, not patched)
- Only `ndshape` and `ndcopy` use `[patch.crates-io]`

### Key Discoveries (zero-usage items eliminated):
- `BorderRadius`, `AmbientLight`, `clear_children`/`remove_children`/`remove_child`, `LineHeight`, `compute_normals` — **zero usages**, no changes needed
- `LocalTimeline` — 14 system params + 5 test usages, all identical pattern
- `AnimationTarget` — 1 insertion site in [animation.rs:488-516](crates/sprite_rig/src/animation.rs#L488-L516)
- `AnimationEventTrigger` — 2 sites in [animset.rs:124-139](crates/sprite_rig/src/animset.rs#L124-L139)
- `Mesh::insert_attribute`/`insert_indices` — 1 site in [meshing.rs:49-56](crates/voxel_map_engine/src/meshing.rs#L49-L56)
- `NextState::set` — 8 production + 16 test calls
- `PhysicsInterpolationPlugin`/`IslandSleepingPlugin` — 1 site in [lib.rs:272-278](crates/protocol/src/lib.rs#L272-L278)
- Web crate uses `default-features = false` and is missing input features

## Desired End State

All dependencies at Bevy 0.18 versions. `cargo check-all` passes. `cargo test-all` passes. `cargo server` and `cargo client` run and load all `.ron` assets without errors.

## What We're NOT Doing

- Adopting any new Bevy 0.18 features (per-camera AmbientLight, etc.)
- Refactoring beyond what's required for compilation
- Updating vendored git/ sources (they're excluded from workspace)

## Phase 2: Lightyear Code Changes

### Overview
Migrate `LocalTimeline` from Component to Resource across 14 system parameters and 5 test usages.

### Changes Required:

#### 1. System Parameter Migration (14 sites)
**Files**: `crates/protocol/src/ability.rs`, `crates/protocol/src/hit_detection.rs`, `crates/server/src/gameplay.rs`

In every system signature, change:
```rust
// Before:
timeline: Single<&LocalTimeline, Without<ClientOf>>,
// After:
timeline: Res<LocalTimeline>,
```

The `.tick()` call remains unchanged. Remove any `Without<ClientOf>` import if it becomes unused.

**Specific locations** (all use identical pattern):
- `crates/protocol/src/ability.rs` — lines 685, 777, 797, 1001, 1211, 1290, 1363, 1396, 1505, 1520 (10 systems)
- `crates/protocol/src/hit_detection.rs` — lines 78, 151 (2 systems)
- `crates/server/src/gameplay.rs` — lines 116, 156 (2 systems)

#### 2. Test Infrastructure Migration
**File**: `crates/protocol/tests/ability_systems.rs`

The `spawn_timeline` helper (line 188) spawns `LocalTimeline` as a component. Change to insert it as a resource:

```rust
// Before:
fn spawn_timeline(world: &mut World, tick_value: u16) -> Entity {
    let entity = world
        .spawn((LocalTimeline::default(), PreSpawnedReceiver::default()))
        .id();
    let mut timeline = world.get_mut::<LocalTimeline>(entity).unwrap();
    timeline.apply_delta(TickDelta::from_i16(tick_value as i16));
    entity
}

// After:
fn insert_timeline(world: &mut World, tick_value: u16) {
    world.insert_resource(LocalTimeline::default());
    let mut timeline = world.resource_mut::<LocalTimeline>();
    timeline.apply_delta(TickDelta::from_i16(tick_value as i16));
}
```

The `advance_timeline` helper (line 196) and all test assertions (lines 259, 320) also change from entity component access to resource access:

```rust
// Before:
fn advance_timeline(world: &mut World, timeline_entity: Entity, delta: i16) {
    let mut timeline = world.get_mut::<LocalTimeline>(timeline_entity).unwrap();
    timeline.apply_delta(TickDelta::from_i16(delta));
}
// After:
fn advance_timeline(world: &mut World, delta: i16) {
    let mut timeline = world.resource_mut::<LocalTimeline>();
    timeline.apply_delta(TickDelta::from_i16(delta));
}

// Assertions — Before:
let timeline = app.world().get::<LocalTimeline>(timeline_entity).unwrap();
// After:
let timeline = app.world().resource::<LocalTimeline>();
```

Update all call sites that pass `timeline_entity` to these helpers.

#### 3. Physics Plugin Compatibility

Check if `PhysicsInterpolationPlugin` and `IslandSleepingPlugin` still exist in avian3d 0.5. If `PhysicsInterpolationPlugin` was removed (lightyear 0.26 release notes say it's "obsolete"), remove that `.disable::<PhysicsInterpolationPlugin>()` line. ## Phase 3: Bevy API Changes

### Overview
Fix remaining Bevy 0.18 breaking API changes: AnimationTarget split, AnimationEventTrigger rename, Mesh try_* methods, NextState behavior.

### Changes Required:

#### 1. AnimationTarget → AnimationTargetId + AnimatedBy
**File**: `crates/sprite_rig/src/animation.rs`

Update import (line 5):
```rust
// Before:
use bevy::animation::{animated_field, AnimationTarget, AnimationTargetId};
// After:
use bevy::animation::{animated_field, AnimatedBy, AnimationTargetId};
```

Update insertion (lines 508-512):
```rust
// Before:
commands.entity(bone_entity).insert(AnimationTarget {
    id: target_id,
    player: entity,
});
// After:
commands.entity(bone_entity).insert((
    AnimationTargetId(target_id),
    AnimatedBy(entity),
));
```

Note: `AnimationTargetId::from_names()` usage at lines 233, 265, 289, 318, 350 should remain valid — it already produces an `AnimationTargetId`. Verify at compile time.

#### 2. AnimationEventTrigger Field Rename
**File**: `crates/sprite_rig/src/animset.rs`

Line 128 — construction:
```rust
// Before:
AnimationEventTrigger {
    animation_player: entity,
}
// After:
AnimationEventTrigger {
    target: entity,
}
```

Line 139 — field access:
```rust
// Before:
let player_entity = trigger.trigger().animation_player;
// After:
let player_entity = trigger.trigger().target;
```

#### 3. Mesh Method Renames
**File**: `crates/voxel_map_engine/src/meshing.rs:49-56`

```rust
// Before:
mesh.insert_attribute(Mesh::ATTRIBUTE_POSITION, positions);
mesh.insert_attribute(Mesh::ATTRIBUTE_NORMAL, normals);
mesh.insert_attribute(Mesh::ATTRIBUTE_UV_0, tex_coords);
mesh.insert_indices(Indices::U32(indices));

// After:
mesh.try_insert_attribute(Mesh::ATTRIBUTE_POSITION, positions).expect("valid position attribute");
mesh.try_insert_attribute(Mesh::ATTRIBUTE_NORMAL, normals).expect("valid normal attribute");
mesh.try_insert_attribute(Mesh::ATTRIBUTE_UV_0, tex_coords).expect("valid uv attribute");
mesh.try_insert_indices(Indices::U32(indices)).expect("valid indices");
```

Per project rules: fail loudly with `expect()`, never silently ignore errors.

#### 4. NextState::set → set_if_neq
**Files**: `crates/protocol/src/app_state.rs`, `crates/ui/src/lib.rs`, `crates/client/src/map.rs`, plus test files

Replace all `next_state.set(...)` with `next_state.set_if_neq(...)` to preserve Bevy 0.17 behavior (no re-entry triggers). This is the safest mechanical migration.

**Production code (8 calls):**
- `crates/protocol/src/app_state.rs:46`

## Risk Items

1. **RON 0.9 → 0.12** (HIGH) — 2 major version jump. If `.ron` files fail to parse, manual syntax fixes required. No automated migration tool exists.
2. **PhysicsInterpolationPlugin removal** (LOW) — May or may not exist in avian 0.5. Compiler will tell us.
3. **Tick fixed-point types** (LOW) — Lightyear 0.26 uses `U0F16`/`U16F16`/`I16F16` for tick math. Our code only calls `timeline.tick()` and `TickDelta::from_i16()`, which should still work. Watch for compile errors.
4. **ReplicationGroup default change** (LOW) — Default group ID 0 instead of per-entity. Our `Replicate::to_clients(NetworkTarget::All)` usage should be fine unless we rely on same-message delivery for related entities.
