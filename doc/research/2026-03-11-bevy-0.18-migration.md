# Research: Bevy 0.18 Migration

**Date**: 2026-03-11T07:49:46-07:00
**Git Commit**: 034969655658529df1cb04b08544cd134af5c773
**Branch**: master

## Research Question

How to update Bevy to 0.18, including bumping lightyear to 0.26, avian3d to 0.5, leafwing-input-manager to 0.20, bevy_common_assets to 0.15, and any other affected dependencies.

## Summary

The migration involves 6 dependency version bumps and ~15 code changes across the codebase. Most breaking changes come from Bevy 0.18 itself (64 total breaking changes, ~15 affect this project) and Lightyear 0.26 (5 breaking changes, ~3 affect this project). Avian 0.5, leafwing 0.20, and bevy_common_assets 0.15 are pure Bevy-compat bumps with no library-specific API changes.

The **highest-risk change** is the `ron` version bump from 0.9 to 0.12, which lightyear 0.26 requires. This may cause RON deserialization changes in `.ron` asset files.

## Breaking Changes That Affect This Project

### 1. Lightyear: `LocalTimeline` is now a Resource (was Component)

**Impact: 4+ files**

`LocalTimeline` changed from a query component (`Single<&LocalTimeline, Without<ClientOf>>`) to a resource (`Res<LocalTimeline>`).

Affected locations:
- `crates/server/src/gameplay.rs:117` ‚Äî `check_death_and_respawn`
- `crates/server/src/gameplay.rs:157` ‚Äî `expire_invulnerability`
- `crates/protocol/src/hit_detection.rs:78-79,100` ‚Äî `process_hitbox_hits`
- `crates/protocol/src/hit_detection.rs:151,171` ‚Äî `process_projectile_hits`
- `crates/protocol/src/ability.rs` ‚Äî ability systems using `LocalTimeline`

**Migration**: Change `Single<&LocalTimeline, Without<ClientOf>>` ‚Üí `Res<LocalTimeline>` in all system signatures. Remove `Without<ClientOf>` filter. Access remains `timeline.tick()`.

### 2. Lightyear: Tick math types changed to fixed-point

**Impact: Depends on usage**

- `Overstep` now uses `U0F16` instead of `f32`
- `TickInstant` uses `U16F16` fixed-point
- `TickDelta` uses `I16F16` signed fixed-point
- New dependency on the `fixed` crate

**Migration**: Check any code that does arithmetic with `Tick`, `TickInstant`, or `TickDelta` values. The project uses `timeline.tick()` which returns a `Tick` ‚Äî if only used for comparisons and storage, this may not require changes. If used in arithmetic with `f32`, will need conversion.

### 3. Lightyear: `ReplicationGroup` default behavior changed

**Impact: Low ‚Äî check server entity spawning**

Entities without explicit `ReplicationGroup` now use default group ID `0` instead of per-entity IDs. Default group no longer guarantees same-message delivery for related entities.

**Migration**: Review `crates/server/src/gameplay.rs:66,204` where `Replicate::to_clients(NetworkTarget::All)` is used. If entity bundles with multiple components need atomic replication, add explicit `ReplicationGroup`.

### 4. Bevy: `AnimationEventTrigger` field rename

**Impact: 1 file**

`animation_player` field renamed to `target`.

Affected: `crates/sprite_rig/src/animset.rs:128` ‚Äî uses `AnimationEventTrigger`

### 5. Bevy: `AnimationTarget` split into `AnimationTargetId` + `AnimatedBy`

**Impact: sprite_rig animation code**

```rust
// 0.17
entity.insert(AnimationTarget { id: AnimationTargetId(id), player: player_entity });
// 0.18
entity.insert((AnimationTargetId(id), AnimatedBy(player_entity)));
```

Affected: `crates/sprite_rig/src/animation.rs` ‚Äî animation graph/player setup

### 6. Bevy: Mesh `try_*` function renames

**Impact: voxel_map_engine**

Nearly all `Mesh` methods renamed to `try_*` variants returning `Result`. Key renames:
- `insert_attribute` ‚Üí `try_insert_attribute`
- `insert_indices` ‚Üí `try_insert_indices`
- `indices` ‚Üí `try_indices`
- `compute_normals` ‚Üí `try_compute_normals`

Affected: `crates/voxel_map_engine/src/meshing.rs` ‚Äî mesh construction for voxel chunks

### 7. Bevy: `ron` no longer re-exported from bevy

**Impact: Already mitigated**

Project already has direct `ron = "0.9"` workspace dependency. But needs bump to `0.12` for lightyear 0.26 compat. See risk item below.

### 8. Bevy: State transitions always trigger `OnExit`/`OnEnter`

**Impact: UI state management**

`next_state.set(State::Menu)` now triggers `OnExit`/`OnEnter` even if already in that state. Use `set_if_neq()` for old behavior.

Affected: `crates/ui/src/lib.rs` ‚Äî 6 `NextState` mutations at lines 122, 134, 225, 306, 430, 482

**Migration**: Audit all `next_state.set()` calls. Replace with `set_if_neq()` where re-entering the same state would cause problems (e.g., despawning/respawning UI).

### 9. Bevy: Input source feature gating

**Impact: web crate**

Input sources (mouse, keyboard, gamepad, touch) now behind cargo features. When using `default-features = false`, must explicitly enable needed input features.

Affected: `crates/web/Cargo.toml:15-35` ‚Äî explicit WASM feature set with `default-features = false`

**Migration**: Add `"mouse"`, `"keyboard"`, `"gamepad"` features to web crate's bevy dependency.

### 10. Bevy: Entity method renames (`clear_children` ‚Üí `detach_all_children`, etc.)

**Impact: Search codebase**

- `clear_children` ‚Üí `detach_all_children`
- `remove_children` ‚Üí `detach_children`
- `remove_child` ‚Üí `detach_child`
- `clear_related` ‚Üí `detach_all_related`

**Migration**: Grep for old method names and rename.

### 11. Bevy: `BorderRadius` moved to `Node` field

**Impact: UI code**

```rust
// 0.17
entity.insert(BorderRadius::all(Val::Px(10.0)));
// 0.18
node.border_radius = BorderRadius::all(Val::Px(10.0));
```

Affected: Check `crates/ui/src/lib.rs` for any `BorderRadius` usage.

### 12. Bevy: `AssetPlugin` new field

**Impact: 3 files**

New `use_asset_processor_override` field. Use `..Default::default()`.

Affected:
- `crates/server/src/main.rs:17`
- `crates/client/src/main.rs:33`
- `crates/protocol/tests/physics_isolation.rs:14`

**Migration**: Ensure all `AssetPlugin` construction uses `..Default::default()` or `..default()` (likely already does).

### 13. Bevy: `LineHeight` separate component

**Impact: Check UI**

`line_height` removed from `TextFont`. `LineHeight` is now a required component on `Text`/`Text2d`/`TextSpan`.

**Migration**: If any `TextFont` in `crates/ui/src/lib.rs` sets `line_height`, extract it to a separate `LineHeight` component.

### 14. Bevy: `AmbientLight` ‚Üí `GlobalAmbientLight` resource

**Impact: Check rendering**

`AmbientLight` resource renamed to `GlobalAmbientLight`. `AmbientLight` is now a per-camera component.

**Migration**: Grep for `AmbientLight` resource usage and rename.

### 15. Bevy: Feature rename `animation` ‚Üí `gltf_animation`

**Impact: Possibly web crate**

Check if `bevy_animation` feature (used by web and sprite_rig crates) is affected. The rename is for the top-level `animation` convenience feature, not the `bevy_animation` subcrate feature ‚Äî so likely no impact.

## RON 0.9 ‚Üí 0.12 Migration Risk

Lightyear 0.26 depends on `ron 0.12`. The project currently uses `ron 0.9`. This is a **2 major version jump** and the highest-risk item.

### Known RON breaking changes (0.9 ‚Üí 0.10 ‚Üí 0.11 ‚Üí 0.12):

RON 0.10 was a major rewrite. Key changes:
- **Implicit `Some` wrapping**: RON 0.10+ can implicitly wrap values in `Some` for `Option<T>` fields
- **Struct naming changes**: May affect how newtype structs are (de)serialized
- **Integer/float parsing changes**: Stricter parsing in some cases
- **Extension syntax changes**: `#![enable(...)]` syntax may have changed

### Affected `.ron` asset files:

All files under `assets/` that use RON format:
- `assets/*.ability.ron` ‚Äî ability definitions
- `assets/*.ability_slots.ron` ‚Äî ability slot configs
- `assets/*.ability_manifest.ron` ‚Äî ability manifests
- `assets/anims/humanoid/*.anim.ron` ‚Äî animation definitions
- `assets/anims/humanoid/*.animset.ron` ‚Äî animation set definitions
- `assets/anims/humanoid/*.rig.ron` ‚Äî rig definitions

**Migration**: After bumping `ron` to 0.12, run the game and verify all `.ron` assets load correctly. Fix any deserialization errors.

## Recommended Migration Order

1. **Cargo.toml version bumps** (all at once ‚Äî they're interdependent)
2. **`cargo check-all`** to get the full error list
3. **Fix compilation errors** guided by the breaking changes above
4. **Verify `.ron` asset loading** at runtime (`cargo server`, `cargo client`)
5. **Run tests** to catch behavioral regressions (especially state transition changes)
